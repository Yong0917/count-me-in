# count-me-in 개발 로드맵

> 마지막 업데이트: 2026-06-11
> 버전: v1.0
> 기준: PRD v0.1 (MVP) · 개발 인력 1인 풀스택

## 📋 프로젝트 개요

참가 코드 하나로 같은 페이지에 모여, 달력에서 오늘 누가 오는지 바로 확인하고 내 참석을 탭 한 번으로 표시하는 소모임용 참석 공유 웹앱이다. 회원가입 없이 코드 입력(또는 링크 탭) → 이름 선택만으로 사용하며, 5명 내외 소모임에 맞춘 최소 기능을 지향한다. 타깃은 모임 일정을 등록하는 주최자 1명과 빠른 참석 표시가 필요한 멤버 4~5명이다.

## 🎯 성공 지표 (KPI)

| 지표 | 목표 |
| --- | --- |
| 모임당 참석 응답률 (멤버 중 응답한 비율) | ≥ 80% |
| 링크 접속 → 참석 표시까지 소요 시간 | ≤ 30초 |
| 주 1회 이상 앱을 여는 멤버 비율 | ≥ 70% |
| 모임 전날 기준 "미정" 비율 | ≤ 20% |
| 초기 화면 로딩 체감 시간 | ≤ 2초 |

## 🏗️ 기술 스택

| 영역 | 선택 | 이유 |
| --- | --- | --- |
| 프론트엔드 | Next.js (App Router) + React + TypeScript | PRD 지정. 서버 라우팅/배포 용이, 타입 안정성 |
| 스타일 | Tailwind CSS | 모바일 우선 반응형을 빠르게 구현 |
| 달력 UI | 경량 커스텀 월간 그리드 | 외부 의존 최소화, MVP 규모에 충분 |
| 백엔드/DB | Supabase (PostgreSQL) | 관리형 DB + Supabase JS 직접 접근, 인프라 최소화 |
| 데이터 접근 통제 | `share_token` 기반 RLS + RPC/Edge Function | 정식 인증 없이 그룹 격리 |
| 배포 | Vercel (FE) + Supabase (DB) | 환경변수로 URL/anon key 주입 |
| 타임존 | Asia/Seoul 고정 | 한국 사용자 한정 (가정) |

## 📅 개발 로드맵

> 일정은 1인 풀스택 순차 진행 기준 추정치이며, 역할 구분 없이 우선순위로만 표기한다.
> 전체 추정: 약 4~5주 (Phase 0~4 합산).

**완료 기준**: 로컬과 Vercel에서 빈 앱이 구동되고, Supabase에 전체 스키마가 적용되어 있다.

- [x] 디렉토리 구조/린트/포매터 등 기본 컨벤션 설정 | 예상: 0.5d | 우선순위: 🟡중간 ✅ src/, @/* alias, ESLint

**완료 기준**: 코드 입장 → 이름 선택 → 달력(디폴트=오늘) 흐름이 모바일에서 동작한다.
**대상 기능**: F1, F2, F4

## 🔗 기술적 의존성

```
Phase 0 (셋업 · 스키마 · Supabase 클라이언트)
   │
   ├─ F1 (그룹 생성/코드·토큰 발급)
   │      │
   │      └─ F2 (코드 입장 · 이름 선택 · localStorage)
   │             │
   │             └─ F4 (월간 달력 · 날짜 선택 상태)
   │                    │
   │                    ├─ F5 (참석 토글) ──┐
   │                    │                   ├─ F6 (현황 패널 · 달력 마커)
   │                    └─ F3-비정기 일정 ──┘
   │                           │
   │                           └─ F3-정기 + Lazy materialize ── F7 (메모)
   │
   └─ RLS/share_token 격리 (Phase 0 골격 → Phase 4 마감)
```

- 핵심 의존: 코드/이름 식별(F1→F2)이 없으면 달력·참석 전부 막힘 → 최우선.
- F6(현황)은 F5(토글)와 F3(일정)에 동시 의존 → 같은 Phase에 묶음.
- 정기 일정/메모는 비정기 일정·참석 구조가 선행되어야 함 → Phase 3.
